
This file contains two parts:

1. **Structured metadata** written in YAML.
2. **Narrative explanation** written in Markdown.

The metadata describes the technical aspects of the reconstruction:

- location
- time period
- map calibration
- referenced assets

The Markdown section contains the Archivist’s explanation of the reconstruction.

Because the file is plain text, it can be edited in any editor and stored in version control.

---

## Cultural Orientation

The system is intentionally aligned with cultural traditions that preserved history through layered commentary and archival record keeping.

The design therefore prioritizes:

- long-term textual durability
- multilingual capability
- compatibility with Hebrew and Japanese text
- archivist-centered workflows

This differs from systems designed primarily around Western corporate or government data models.

---

## Reproducible Historical Debate

The ultimate goal of the system is not only reconstruction, but **debate**.

When multiple entries exist for the same location across time, they can be compared by:

- historians
- researchers
- AI analysis tools

Because each entry includes both the narrative and the assets used to produce it, disagreements can be evaluated against the recorded evidence rather than only rhetorical argument.

This makes it significantly harder to alter historical claims without leaving a trace in the reconstruction.

---

## Scope of the System

The system is intentionally modest in scope.

It is not intended to replace GIS systems or professional simulation tools. Instead, it provides a lightweight framework for:

- historical exploration
- speculative reconstruction
- educational storytelling
- cultural documentation

Over time, more sophisticated tools may be built on top of the framework, but the core structure will remain simple and transparent.

---

## Current Status

The system is currently in an early development stage. The core workflow already supports:

- entry creation
- basemap import
- automated geometry generation
- Blender editing
- ThreeJS visualization

Future development will focus on improving stability, documentation, and automated testing.
